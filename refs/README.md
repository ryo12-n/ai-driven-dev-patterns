# refs/ — 外部参照資料

## 目的

トリアージや開発セッション中に参照する外部資料（記事・ドキュメント・ツール情報など）を格納する。

## 運用ルール

- 外部参照を追加する際は `inbox/ref_<名前>.md` にポインターファイルを作成する
  - ポインターファイルには資料の概要・URL・今後の扱い方針を記載する
- トリアージ時に `inbox/ref_*.md` を走査し、以下を判断する
  - refs/ への格納が必要 → refs/ に本体を配置し、ポインターを削除
  - 一時的な参照で不要 → ポインターを削除
  - 判断保留 → 次回トリアージに持ち越し
- refs/ の本体ファイルはトリアージで削除しない（ポインターのみ削除可）
- 孤立エントリ（対応するポインターが存在しない refs/ ファイル）はトリアージで検出・確認する

## ファイル命名規則

```
refs/<カテゴリ>-<名前>.md
```

例:
- `refs/tool-claude-code-hooks.md`
- `refs/article-ai-dev-patterns.md`
