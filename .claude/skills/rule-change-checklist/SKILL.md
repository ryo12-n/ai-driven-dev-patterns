# ルール変更時の連動更新チェックリスト

> **このファイルの位置づけ**
> ルール変更を含む施策（`.claude/skills/` や `.claude/rules/` や `.claude/agents/` 配下のファイルを変更する施策）で、
> 連動更新の漏れを防ぐための標準チェックリスト。
> dev_manager がタスクを含める際、または専門ロールが作業完了前に確認する際に使用する。

---

## 標準タスクパターン: 調査 → 文書化 → ルール反映 → 連動更新

ルール変更を含む施策では、以下の定型パターンに従う。

```
1. 調査        — 現状の参照関係・影響範囲を把握する
2. 文書化      — 設計・ガイドライン文書を作成する（必要な場合）
3. ルール反映  — .claude/skills/ や .claude/rules/ や .claude/agents/ のファイルを変更する
4. 連動更新    — 以下のチェックリストに従い、関連ファイルを更新する
```

---

## チェックリスト

### 1. 関連ファイル一覧の確認

変更したファイルに「関連ファイル一覧」セクションがある場合、そのテーブルに記載された全ファイルについて連動更新の要否を確認する。

- [ ] 変更ファイルの「関連ファイル一覧」セクションを確認した
- [ ] 一覧に記載された各ファイルについて、連動更新の要否を判断した
- [ ] 連動更新が必要なファイルを更新した

### 2. docs/ の更新

`.claude/skills/` や `.claude/rules/` や `.claude/agents/` 配下のルール定義が正の情報源（Source of Truth）であり、`docs/` 配下のドキュメントはその人間向け可視化である。ルールファイルを変更した場合は、関連する `docs/` のドキュメントも合わせて更新する。

- [ ] 変更内容が `docs/` のどのドキュメントに影響するかを特定した
- [ ] 該当するドキュメントを更新した
- [ ] 更新後のドキュメントが変更後のルールと整合していることを確認した

### 3. 品質基準ポリシーの連動更新

ルール変更がトリアージや品質基準のチェック観点に影響する場合、基準文書も更新する。

- [ ] 変更内容が以下のいずれかに該当するかを確認した:
  - ライフサイクルステージの追加・変更・削除
  - 品質ゲートの基準変更
  - 課題起票・ルーティング手段の変更
- [ ] 該当する場合、品質基準に関するドキュメントを更新した

### 4. テンプレートの連動更新

変更がテンプレートファイルの構成に影響する場合、テンプレートも更新する。

- [ ] `sessions/_template/` 配下のテンプレートに影響がないか確認した
- [ ] `sessions/initiatives/_template/` 配下のテンプレートに影響がないか確認した（存在する場合）

### 5. コミットメッセージ規約の確認

新しいセッション種別を追加した場合、コミットメッセージ規約にも反映する。

- [ ] `.claude/rules/commit-message.md` に新セッション種別の追加が必要か確認した

---

## 使い方

### dev_manager（タスク作成時）

ルール変更を含む施策では、実施タスクの最後に以下の固定タスクを追加する:

```markdown
| DT-XXX | ルール変更連動更新チェックリスト（rule-change-checklist スキル）を実施する | チェックリストの全項目が確認済み |
```

### 専門ロール（作業完了前）

ルールファイルを変更した後、コミット前にチェックリストの各項目を確認し、作業ログに確認結果を記録する。

## 関連ファイル一覧

本スキルファイルの内容を変更した場合、以下のファイルの連動更新が必要になる可能性がある。

| ファイル | 連動更新の内容 |
|---------|-------------|
| `.claude/skills/dispatcher/SKILL.md` | 「タスクリストに含める固定タスク」のチェックリスト参照 |

| `docs/` 配下の関連ドキュメント | チェックリスト手順の記述 |
