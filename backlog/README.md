# backlog/

## 用途

具体的な次施策候補の保管場所。トリアージで承認された施策候補を 1 施策 1 ファイル形式で管理する。

dev_manager の知見収集ステップや、セッション完了時のルーティングで「具体的な次アクションとして施策化可能」と判断されたものがここに配置される。

## 記載フォーマット

`_template.md` をコピーして新規ファイルを作成する。

```markdown
# <施策名>

## メタ情報

| 項目 | 内容 |
|------|------|
| **優先度** | 高 / 中 / 低 |
| **ステータス** | 候補 |
| **起票日** | YYYY-MM-DD |

## 課題・背景
<!-- 現状の問題点を記述する -->

## 期待効果
<!-- この施策を実施することで得られる効果を記述する -->

## 補足・参考情報
<!-- 任意。関連する議論・参考リンク・技術的な制約など -->
```

## ファイル命名規則

```
backlog/<施策名>.md
```

例: `backlog/spec作成ガイドラインの整備.md`

## ライフサイクル

1. トリアージ・知見収集ステップで施策候補を発見 → `backlog/<施策名>.md` を作成
2. 施策として開始する際に、内容を `sessions/` 配下の plan.md に引き継ぐ
3. 施策開始後、backlog のファイルを削除（git 履歴が証跡）
