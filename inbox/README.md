# inbox/

## 用途

知見・気づきの一時保管場所（判断保留のバッファ）。

セッション中に発見した気づきのうち、即座にルール化や施策化の判断ができないものをここに一時的に保管する。トリアージセッションで定期的に走査し、適切な振り分け先（backlog / 課題管理.csv / .claude/rules/ / roles/ / 破棄）を決定する。

## 記載フォーマット

1ファイル = 1件の気づき。以下のフォーマットで `.md` ファイルを作成する。

```markdown
# <タイトル>

- **起票日**: YYYY-MM-DD
- **起票元**: <セッション名またはタスクID>
- **種別**: 気づき / 改善提案 / リスク / 質問
- **内容**: 何が分かったか・何を検討すべきか
- **補足**: （あれば）
```

## ファイル命名規則

```
inbox/<タイトルのスラッグ>.md
```

例: `inbox/spec作成時のエッジケース漏れ対策.md`

## ライフサイクル

1. セッション中に気づきを発見 → `inbox/<タイトル>.md` を作成
2. トリアージセッションで走査 → 振り分け先を決定
3. 振り分け実行後、inbox のファイルを `git rm` で削除（git 履歴が証跡）

## 注意事項

- inbox はあくまで一時保管場所。長期間放置しない（トリアージで定期的に処理する）
- 参照物（他 PJ の実装など実体ファイルを持ち込む場合）は `refs/` に本体を置き、`inbox/ref_<名前>.md` にポインターメモを置く
