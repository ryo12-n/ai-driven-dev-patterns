# inbox/

## 用途

**人間のアイデア・気づきの保管場所**。

人間がセッション中に思いついたアイデアや改善提案を一時的に保管する場所。トリアージセッションで定期的に走査し、適切な振り分け先（backlog / 課題管理.csv / .claude/rules/ / .claude/agents/ / 破棄）を決定する。

### inbox と 課題管理.csv の使い分け

| 起票者 | 保管先 | 用途 |
|--------|--------|------|
| **人間** | `inbox/` | アイデア・改善提案・判断保留の気づき |
| **AI（dev_manager 等）** | `課題管理.csv` | セッション中に発見した課題・知見・再発パターン |

AI がセッション中に発見した知見は、dev_manager の知見収集ステップ（report.md）でルーティング先を判断し、`課題管理.csv` またはロール定義・ルールファイルに直接反映する。判断保留の気づきのみ `inbox/` に配置する。

## 記載フォーマット

1ファイル = 1件の気づき。以下のフォーマットで `.md` ファイルを作成する。

```markdown
# <タイトル>

- **起票日**: YYYY-MM-DD
- **起票元**: <セッション名またはタスクID>
- **種別**: 気づき / 改善提案 / リスク / 質問
- **内容**: 何が分かったか・何を検討すべきか
- **補足**: （あれば）
```

## ファイル命名規則

```
inbox/<タイトルのスラッグ>.md
```

例: `inbox/spec作成時のエッジケース漏れ対策.md`

## ライフサイクル

1. 人間がアイデア・気づきを発見 → `inbox/<タイトル>.md` を作成
2. トリアージセッションで走査 → 振り分け先を決定
3. 振り分け実行後、inbox のファイルを `git rm` で削除（git 履歴が証跡）

## 注意事項

- inbox はあくまで一時保管場所。長期間放置しない（トリアージで定期的に処理する）
- 参照物（他 PJ の実装など実体ファイルを持ち込む場合）は `refs/` に本体を置き、`inbox/ref_<名前>.md` にポインターメモを置く
- AI の知見は原則 `課題管理.csv` にルーティングする。inbox への AI 起票は判断保留のケースに限定する
