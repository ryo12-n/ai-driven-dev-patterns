## ADDED Requirements

### Requirement: 注文の確定
API はカートの内容に基づいて注文を確定するエンドポイントを提供しなければならない（MUST）。
注文確定時には在庫チェックを実施しなければならない（MUST）。

#### Scenario: 正常な注文確定
- **GIVEN** カート `{cart_id}` に1件以上の商品が存在する
- **AND** カート内の全商品について十分な在庫が存在する
- **WHEN** クライアントが `POST /orders` に `cart_id` を送信する
- **THEN** API は在庫を確認し、注文レコードを作成しなければならない（MUST）
- **AND** 各商品の在庫数を注文数量分だけ減算しなければならない（MUST）
- **AND** カートを空にしなければならない（MUST）
- **AND** 作成された注文の詳細（注文ID・商品リスト・合計金額・ステータス）を JSON で返さなければならない（MUST）
- **AND** HTTP ステータスコード 201 を返さなければならない（MUST）

#### Scenario: 在庫不足による注文失敗
- **GIVEN** カート `{cart_id}` に商品が存在する
- **AND** カート内の少なくとも1件の商品について在庫が不足している
- **WHEN** クライアントが `POST /orders` に `cart_id` を送信する
- **THEN** API は注文を作成してはならない（MUST NOT）
- **AND** 在庫が不足している商品のリストをエラーレスポンスに含めなければならない（MUST）
- **AND** 在庫の減算を行ってはならない（MUST NOT）
- **AND** HTTP ステータスコード 409 を返さなければならない（MUST）

#### Scenario: 空のカートでの注文
- **GIVEN** カート `{cart_id}` に商品が1件も存在しない
- **WHEN** クライアントが `POST /orders` に `cart_id` を送信する
- **THEN** API はエラーメッセージを JSON で返さなければならない（MUST）
- **AND** HTTP ステータスコード 400 を返さなければならない（MUST）

#### Scenario: 存在しないカートでの注文
- **GIVEN** カート `{cart_id}` が存在しない
- **WHEN** クライアントが `POST /orders` に `cart_id` を送信する
- **THEN** API はエラーメッセージを JSON で返さなければならない（MUST）
- **AND** HTTP ステータスコード 404 を返さなければならない（MUST）

---

### Requirement: 注文の在庫チェック整合性
注文確定処理において、在庫チェックと在庫減算は不可分な操作として実行されなければならない（MUST）。

#### Scenario: 同時注文時の在庫整合性
- **GIVEN** プロダクト `{product_id}` の在庫が残り1個である
- **AND** 2つのカートがそれぞれ同商品を1個ずつ含んでいる
- **WHEN** 2つの注文確定リクエストがほぼ同時に送信される
- **THEN** 先に処理された注文のみが成功しなければならない（MUST）
- **AND** 後の注文は在庫不足エラーを返さなければならない（MUST）
- **AND** 在庫数がマイナスになってはならない（MUST NOT）
