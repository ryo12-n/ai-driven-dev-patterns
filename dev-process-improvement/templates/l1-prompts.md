# L1（管理）セッション用プロンプト集

> これらのプロンプトをコピーして L1 セッションの冒頭に貼り付けてください。
> `（）`内は実際の値に置き換えてください。

---

## 1. セッション起動プロンプト

```
あなたはこのディレクトリ（dev-process-improvement/）の管理AIです。
以下のファイルを読み込んでください：
- CLAUDE.md
- .claude/rules/l1-manager.md
- backlog/ideas.md

現在のバックログを確認し、優先度の高い施策候補を3件ピックアップしてください。
```

---

## 2. 施策の立ち上げ指示

```
以下の施策をバックログから選定して立ち上げます。

施策ID: （B001 など）
施策名: （タイトル）

以下を実施してください：
1. initiatives/_template/ を initiatives/（施策フォルダ名）/ にコピーする手順を示す
2. 00_proposal.md の草案を生成する
3. 01_plan.md の草案を生成する
4. 02_tasks.md のタスクリストを生成する（タスクは1件あたり2時間以内で完了できる粒度にする）
```

---

## 3. L2-worker へのキックオフ指示

```
以下の施策の実施フェーズを開始します。
L2-worker セッションに渡す起動プロンプトを生成してください。

施策フォルダ: initiatives/（施策フォルダ名）/
担当フェーズ: フェーズ（番号）
タスク範囲: T（開始番号） 〜 T（終了番号）
```

---

## 4. ゲートレビュー実施

```
以下の施策のゲートレビューを実施します。

施策フォルダ: initiatives/（施策フォルダ名）/
読み込むファイル:
- 01_plan.md（成功基準の確認）
- 06_eval_report.md（評価結果）
- 07_issues.md（未解消課題の確認）

08_gate_review.md に判定結果を記録してください。
```

---

## 5. バックログ整理

```
backlog/ideas.md を読み込み、以下を実施してください：
1. 「candidate」ステータスの施策を優先度順に並べ替える（理由を添えて）
2. 「idea」ステータスで3ヶ月以上経過しているものを確認し、deferred候補を挙げる
3. 整理後のバックログを更新する
```
