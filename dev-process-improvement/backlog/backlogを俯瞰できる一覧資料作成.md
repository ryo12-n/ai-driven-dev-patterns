# backlogを俯瞰できる一覧資料作成

## メタ情報

| 項目 | 内容 |
|------|------|
| **優先度** | 中 |
| **ステータス** | 候補 |
| **起票日** | 2026-03-05 |

## 課題・背景

現在 backlog は 1施策1ファイル形式で管理されているが、施策の全体像を俯瞰する手段がない。施策数が増えるにつれ、優先度やステータスの把握にファイルを個別に開く必要があり、意思決定の効率が低下する。

## 期待効果

- backlog 全体の優先度・ステータスを一覧で把握でき、施策選定の判断が迅速になる
- 施策間の関連性や重複を発見しやすくなる
- トリアージ時の判断材料として活用できる

## 補足・参考情報

- 一覧資料の形式（CSV、Markdown テーブル、自動生成スクリプト等）は施策開始時に検討する
- `課題管理.csv` との関係整理も必要（backlog と課題管理の役割分担の明確化）
