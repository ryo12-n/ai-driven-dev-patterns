## 課題・背景

`--worktree` を使った隔離開発の組み込み方法が未検討。AIに任せた変更を本流から安全に分離しつつ、並列開発しやすくするメカニズムとして有望。

## 期待効果

並列開発・変更分離を安全に運用できる標準パターンを確立する。

## 補足・参考情報

元ネタ: `inbox/gitworktreeによるクロードの開発を標準として採用できるか検討する.md`
協調プロトコル整備（commit/lock/test）施策と連携可能だが、スコープは異なる（worktree はインフラ的な隔離メカニズム）。
