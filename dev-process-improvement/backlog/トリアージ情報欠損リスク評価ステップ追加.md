## 課題・背景

トリアージ走査の結果、inbox ファイルの削除・統合・施策化を行う際に、元ファイルに含まれる詳細な知見・参照リンクが失われるリスクがある。

20260306 トリアージで実際に発生した事例:
- `inbox/長文のrulesの扱い.md`: 14件のナレッジポイントと15件の参照リンクを含んでおり、「既存 backlog に統合」の方針では情報が欠損する見込みだった → ユーザー指摘で新規 backlog として施策化に方針変更
- `inbox/claude_rules配下の認識改める.md`: 6件の参照リンクが既存 backlog に統合時に欠損する可能性があった → 知見セクションとして追記する形で対応

現在の triage-worker は走査・分類を行うが、削除・統合候補の情報欠損リスクを体系的に評価するステップがない。

## 期待効果

- inbox 処理時の情報欠損を防止する体系的なチェック機構の導入
- 削除・統合・施策化の判断品質の向上
- 専用のワーカー＋評価者セットによる独立した品質保証

## 想定する実装案

### 案: 情報欠損リスク評価用ワーカーセットの追加

通常の走査ワーカーセット完了後、削除・統合候補が出た段階で追加セットをディスパッチする。

**情報欠損リスク評価ワーカーの責務:**
1. 削除候補の各ファイルについて、含まれる知見・参照リンク・具体的データを棚卸し
2. 統合候補について、統合先に移行される情報と消失する情報を対比
3. 施策化候補について、backlog テンプレートに収まらない詳細情報の保全方法を提案
4. リスク評価テーブルの作成（ファイル名・含有知見数・消失リスク・推奨アクション）

**情報欠損リスク評価エバリュエーターの責務:**
1. ワーカーのリスク評価の妥当性を検証
2. 見落としている知見・参照の有無を確認
3. 推奨アクションの適切性を評価

### 作業フロー変更イメージ

```
既存: set-1(走査) → set-2(走査) → レポート集約 → アクション実施 → PR
変更: set-1(走査) → set-2(走査) → set-3(情報欠損リスク評価) → レポート集約 → アクション実施 → PR
```
