## 課題・背景

北村さんの先進的な AI 開発プラクティスのエッセンスを抽出し、ai-driven-dev-patterns に統合可能な要素を特定・導入したい。

### 主要な取り組み内容

1. **エージェントの階層化構成**: メインエージェント＋サブエージェントの分業体制。モデルの使い分け（重いタスクは Opus、軽いタスクは軽量モデル）
2. **MCP による外部ツール統合**: Jira・Google Workspace・Slack 等を MCP 経由で AI に直接操作させる
3. **自動テストと AI 多段コードレビュー**: Claude Code + Devin による多段レビュー体制。テスト駆動の安全性担保
4. **夜間バッチでの自動リファクタリング**: 開発時間外に Devin を稼働させ、認知複雑度の改善を自動 PR 化
5. **tmux による 4 並列ターミナル運用**: リモート環境で CLI ベースの並列作業

## 期待効果

- 先進的な AI 開発プラクティスの知見を体系的に評価・取り込みできる
- エージェント階層化・MCP 活用など、既存の仕組みを補完する手法が得られる
