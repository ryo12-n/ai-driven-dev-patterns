# 実施計画: Claude標準ディレクトリ構成のルール化

## フェーズ構成

| フェーズ | 内容 | 成功基準 |
|---------|------|----------|
| 1 | Web調査・標準構成の整理・現状分析・ルールドキュメント作成 | ルールドキュメントが `.claude/rules/claude-directory-guide.md` に配置され、標準構成の説明・現状との差異分析・改善方針が網羅されている |

## 成功基準（全体）

1. `.claude/rules/claude-directory-guide.md` が作成されている
2. Claude Code 公式ドキュメントに基づく標準ディレクトリ構成（`~/.claude/`、`./.claude/`、各サブディレクトリ）が網羅的に記載されている
3. 設定の優先順位（プロジェクトローカル > グローバル > デフォルト）が明記されている
4. 本リポジトリの現状構成と標準構成の差異が分析・記載されている
5. 次施策への改善方針が具体的に提示されている

## リソース・前提条件

- Claude Code 公式ドキュメント（Web検索で取得）
- 本リポジトリの `.claude/` 配下の現状ファイル構成

## リスクと対策

| リスク | 発生確率 | 影響 | 対策 |
|--------|---------|------|------|
| 公式ドキュメントの情報が断片的 | 中 | 中 | 複数ソースを横断調査し、実機の挙動と照合する |
| Claude Code のバージョンアップで構成が変わる | 低 | 中 | 調査時点の日付を明記し、定期的な見直しを想定した構成にする |
