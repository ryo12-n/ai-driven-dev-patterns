# 改善施策提案: セッション前提条件・計画レビュー強制

## 背景・課題

現状、各 L1/L2 セッションにおける「計画の提示・承認・参照情報評価・壁打ち」の実施品質にばらつきがある。

具体的には以下の問題が発生している。

- セッション開始時にブランチ確認・前回申し送り（inbox）の確認・バックログ把握などを行わずに作業に入るケースがある
- L1 が計画を提示する前に実装へ踏み込む、あるいは壁打ちを省略するケースがある
- 「複雑なタスク」かどうかの判断が曖昧なため、確認フェーズがスキップされやすい
- CLAUDE.md の「壁打ち・相互理解確認ルール」と `.claude/rules/l1-manager.md` のルールが統合されておらず、参照元によって適用内容に差がある

この結果、セッションによって計画品質・文脈把握度が不均一になり、手戻りや引き継ぎコストが発生している。

## 目標

1. L1 マネージャーセッション開始時に必ず実施する「前提確認チェックリスト」を定義・強制する
2. 計画策定前の壁打ち・確認フェーズの実施条件を明確化し、省略できない状態にする
3. CLAUDE.md と `.claude/rules/l1-manager.md` の関連ルールの整合を取る

## スコープ

### やること

- `.claude/rules/l1-manager.md` にセッション開始チェックリスト（ブランチ・inbox・バックログ確認）を追加する
- 計画策定前の壁打ち・相互確認を L1 ルールとして明示する（「複雑なタスク」判断基準の具体化を含む）
- CLAUDE.md の「壁打ち・相互理解確認ルール」と l1-manager.md の整合を確認・調整する

### やらないこと

- L2 ワーカー・評価者のセッション品質向上（別施策として検討）
- トリアージセッションのマネージャー・ワーカー構成化（別施策）
- CI/CD や自動チェックの導入
- CLAUDE.md の全体的な見直し（今回はセッション前提確認・計画レビュー箇所に限定）

## 期待される効果

- どの L1 セッションでも一貫した品質の計画策定・承認フローが実施される
- セッション引き継ぎ時の文脈引き継ぎ漏れ（inbox 未読・ブランチ誤り等）が減る
- 壁打ちスキップによる手戻りが減り、ユーザーの確認コストが下がる

## リスク

| リスク | 影響度 | 対策 |
|--------|--------|------|
| チェックリストが形骸化し、形式的に埋めるだけになる | 中 | 各チェック項目に「確認した内容を記録する」形式にして、確認漏れを可視化する |
| ルール追加で l1-manager.md が肥大化し読みにくくなる | 低 | 既存ルールとの統合・整理を同時に行う |
| CLAUDE.md 側の変更で既存セッションの挙動が変わる | 低 | CLAUDE.md の変更は最小限に留め、主に l1-manager.md 側を拡充する |
