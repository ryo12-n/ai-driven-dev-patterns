# 改善施策提案: ロール別エージェントプロンプト ライブラリ整備

## 背景・課題

`refs/ai-driven-development-poc/` には、並列Claudeエージェントチームでアプリ開発を行うパターンが実装されている（feature-builder / bug-fixer / reviewer 等のロール定義）。

現状、`ai-driven-dev-patterns/` 配下で実際のアプリ開発を行う際に利用できるロール定義が存在しない。将来の開発プロジェクトで「マネージャーセッションがサブエージェントとして各ロールを呼び出す」構成を取れるよう、再利用可能なロール定義ライブラリを事前に整備しておきたい。

## 目標

- `ai-driven-dev-patterns/roles/` 配下に、アプリ開発で利用できるロール定義の共通ライブラリを整備する
- poc 参照実装を分析し、本リポジトリのコンテキストに合わせたロール定義（プロンプト）を用意する
- スモールステップで段階的に整備する（まず分析・提案、次に実際のファイル作成）

## 想定アーキテクチャ

```
メインセッション（マネージャー）
  ├── Task tool 経由で各ロールのサブエージェントを呼び出す
  ├── 進捗管理・フェーズゲートチェックを担う
  └── サブエージェント（ワーカー）
        ├── feature-builder（機能実装担当）
        ├── bug-fixer（バグ修正担当）
        ├── reviewer（レビュー担当）
        └── ... （プロジェクトに応じて選択）
```

## スコープ

### フェーズ1（本施策）: 分析と構成案

- poc のロール定義（`refs/ai-driven-development-poc/.claude/`）を分析する
- `ai-driven-dev-patterns/roles/` のディレクトリ構成案と必要ロールリストを作成する
- 分析結果を L1 + ユーザーでレビューし、フェーズ2の実施計画を策定する

### フェーズ2（次施策）: ロール定義ファイルの実装

- フェーズ1の分析結果を踏まえ、実際のロール定義ファイル（プロンプト）を作成する
- `ai-driven-dev-patterns/roles/` 配下に配置する

### やらないこと（本施策）

- `.claude/rules/` 配下の既存ルールファイルの改善
- セッション起動方法・運用手順の整備（フェーズ2以降に検討）
- プロジェクト固有のロール構成の設計（将来のアプリ開発時に対応）

## 期待される効果

- 将来のアプリ開発開始時に、ロール定義を一から作る必要がなくなる
- poc 参照実装の知見が `ai-driven-dev-patterns/` に定着する
- マネージャーセッションが必要なロールを選んでサブエージェントを呼び出せる基盤ができる
