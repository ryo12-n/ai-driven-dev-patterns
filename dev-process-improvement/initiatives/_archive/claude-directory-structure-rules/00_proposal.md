# 改善施策提案: Claude標準ディレクトリ構成のルール化

## 背景・課題

Claude Code には標準的なディレクトリ構成が定められている（`~/.claude/` グローバル設定、`./.claude/` プロジェクト設定）。本リポジトリでは `.claude/rules/`、`.claude/commands/`、`.claude/skills/` を活用しているが、以下の課題がある。

1. Claude Code が認識する標準ディレクトリ構成（各ディレクトリの役割・設定の優先順位・ファイル命名規則等）が明文化されていない
2. 新規プロジェクトに展開する際に、何をどこに配置すべきかの判断基準がない
3. 現状の構成が標準に沿っているか検証されていない

## 目標

1. Claude Code 公式ドキュメントに基づく標準ディレクトリ構成のルールドキュメントを作成する
2. 本リポジトリの現状構成と標準構成の差異を分析し、改善方針を文書化する

## スコープ

### やること

- Claude Code 公式ドキュメント（Web検索）で標準ディレクトリ構成の最新情報を調査する
- 標準構成のルールドキュメントを `.claude/rules/claude-directory-guide.md` として作成する
- 本リポジトリの現状構成と標準構成の差異を分析し、ルールドキュメント内に記載する

### やらないこと

- ディレクトリやファイルの実際の再配置（次施策に持ち越し）
- `roles/` や `docs/` の `.claude/` 配下への移動
- OpenSpec 関連の構成変更（別施策「OpenSpec標準ディレクトリ構成のルール化」のスコープ）

## 期待される効果

- 新規プロジェクト立ち上げ時に `.claude/` の構成判断が即座にできるようになる
- 現状構成の妥当性が検証され、改善の優先順位が明確になる
- チームメンバーが Claude Code の設定体系を正しく理解できる

## リスク

| リスク | 影響度 | 対策 |
|--------|--------|------|
| 公式ドキュメントの情報が不足または変更が頻繁 | 中 | 調査時点のバージョン・日付を明記し、更新可能な構成にする |
| 調査結果と現状構成の乖離が大きく、再配置施策の工数が膨らむ | 低 | 本施策では分析・文書化に留め、実作業は次施策で計画 |

## 壁打ちの背景

フェーズ0の壁打ちで以下を確認した。

- **スコープ決定**: 「Web調査＋ルール明文化のみ」を選択。公式ドキュメントを調査してルールを明文化するが、実際のディレクトリ再配置は次施策に持ち越す方針
- **成果物の配置先**: ルールドキュメントは `.claude/rules/claude-directory-guide.md` に配置する（Claude Code が自動参照する位置）
- **関連施策との切り分け**: OpenSpec 標準構成は別施策として独立。本施策は Claude Code 本体のディレクトリ構成に限定
