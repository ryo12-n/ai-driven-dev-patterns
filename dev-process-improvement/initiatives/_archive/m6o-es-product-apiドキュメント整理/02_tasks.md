## 実施ワーカー（l2-worker）タスク

| ID | タスク | 完了条件 | 優先度 | ステータス |
|----|--------|---------|--------|-----------|
| T-001 | `my_work/` 配下の全ファイルを棚卸しし、一覧表を作成する | `my_work/` 配下の全ファイルパスと内容の概要が一覧化されている | 🔴 | ✅ |
| T-002 | 各ファイルを AI用 / 人間用 / 参考資料 の3区分で分類する（判断軸: 主な利用者が AI か人間か。迷う場合は AI 用に倒す） | 全ファイルに区分が付き、分類根拠が明記されている | 🔴 | ✅ |
| T-003 | 新ディレクトリ構造を設計する（`.claude/rules/`・`.claude/tools/`・`docs/`・`refs/` 等の役割を定義） | 各ディレクトリの用途・配置するファイルの種類が定義された設計書が作成されている | 🔴 | ✅ |
| T-004 | 既存ファイルの移動先マッピング表を作成する（現在パス → 新パス） | 全ファイルに移動先パスが記載されたマッピング表が完成している | 🔴 | ✅ |
| T-005 | 設計成果物を1つのドキュメント（`design.md`）にまとめ、施策ディレクトリ内に保存する | `design.md` に新ディレクトリ構造の定義・分類基準・マッピング表が含まれている | 🟡 | ✅ |
| T-006 | `07_issues.md` の未転記課題を確認し、施策をまたぐ課題を `プロセス改善_課題管理.csv` へ転記する | 全課題に `[転記済 ISS-XXX]` または「転記不要」の判断が付いている | 🔴 | ✅ |
