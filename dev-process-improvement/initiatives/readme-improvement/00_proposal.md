# 改善施策提案: README 改善（起動方法整理・プロンプト方式変更）

## 背景・課題

`dev-process-improvement/README.md` のセッション起動方法に以下の問題がある。

### 課題1: 起動方法の選択肢が多く、整理されていない

起動方法が「方法1〜3」の3択で記載されており、どれを使うべきかが不明確な状態。

- **方法1**: Claude Code Agent Teams（推奨・L1がTaskツールでL2を管理）
- **方法2**: 手動で3セッション起動
- **方法3**: Claude Projects（Web版）

「推奨」と書かれているだけで方法2・3が現役かどうか不明であり、セッション開始時に毎回判断コストが発生している。

### 課題2: 起動プロンプトが長文で、何を渡すべきかが不明確

現状のプロンプトは100字超の長文で、「どのルールに従うか」「どの施策を扱うか」が混在している。
テンプレート（`templates/l1-prompts.md`）も同様に冗長で、コピペして使いにくい状態。

## 目標

1. 起動方法を方法1（Claude Code Agent Teams）に一本化し、方法2・3は削除する
2. 起動プロンプトを「ルールファイル指定 + 施策の指定」だけの短い宣言型に変更する
3. `templates/*.md` を新しいプロンプト形式に対応させる

## 変更後のプロンプト形式（確定）

```
# 新施策開始パターン
.claude/rules/l1-manager.md に従う
backlog: README改善（起動方法整理・プロンプト方式変更）

# 既存施策継続パターン
.claude/rules/l1-manager.md に従う
施策: initiatives/readme-improvement/
```

## スコープ

### やること

- `README.md` の起動方法セクションを方法1のみに書き直す（方法2・3は削除）
- `README.md` の起動プロンプト例を上記の短い宣言型に変更する
- `templates/l1-prompts.md`・`templates/l2-prompts.md` を新しいプロンプト形式に更新する
- `docs/workflow.md` に起動方法の記述がある場合は合わせて更新する

### やらないこと

- `.claude/rules/*.md` のルール内容の変更（別施策で対応）
- Claude Code の設定ファイル（`settings.json`）の変更
- 起動スクリプト（bash ラッパー等）の新規作成

## 期待される効果

- 起動方法の選択に迷わない（1択になる）
- 起動プロンプトが短くなり、施策の切り替えがファイルパスの差し替えだけで済む
- テンプレートが実態に即した内容になり、コピペしてすぐ使える状態になる

## リスク

| リスク | 影響度 | 対策 |
|--------|--------|------|
| 方法2・3を使っていたケースが判明した場合に情報が失われる | 低 | 削除前に git 履歴が残ることを確認済みのため対応不要 |
| 宣言型プロンプトで Claude がルールファイルを読まないケースが発生する | 中 | L2 起動後に `l1-manager.md` が参照されているか確認タスクをフェーズ2に含める |
