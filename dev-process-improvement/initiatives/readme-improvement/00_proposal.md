# 改善施策提案: README 改善（起動方法整理・プロンプト方式変更）

## 背景・課題

`dev-process-improvement/README.md` のセッション起動方法に以下の問題がある。

### 課題1: 起動方法の選択肢が多く、整理されていない

起動方法が「方法1〜3」の3択で記載されており、どれを使うべきかが不明確な状態。

- **方法1**: Claude Code Agent Teams（推奨・L1がTaskツールでL2を管理）
- **方法2**: 手動で3セッション起動
- **方法3**: Claude Projects（Web版）

推奨方式が「推奨」と書かれているだけで、方法2・3が現役かどうか、いつ使うかの判断軸が示されていない。
結果として、セッション開始時に毎回「どれを使えばよいか」の判断コストが発生している。

### 課題2: ルールファイルの指定が口頭指示になっている

現状のプロンプトは「CLAUDE.md と `.claude/rules/l1-manager.md` のルールに従ってください」という自然言語での指示。

- Claude がファイルを実際に読むかどうかは動作に依存しており、確実性がない
- `templates/l1-prompts.md`・`l2-prompts.md` にもルールファイルへの明示的な参照がない
- Claude Code の `--rules` フラグや `@ファイルパス` 記法を使えば、起動時にルールファイルを確実に読み込ませられるが、その仕組みが導入されていない

## 目標

1. README の起動方法セクションを整理し、「いつ何を使うか」を明確にする
2. ルールファイルを Claude Code の機能（`--rules` フラグ等）で確実に読み込ませる起動方式に変更する
3. `templates/*.md` のプロンプトテンプレートを新しい起動方式に対応させる

## スコープ

### やること

- `dev-process-improvement/README.md` の「セッション起動方法」セクションを整理・書き直す
  - 推奨方式を明確化し、各方法の「使うべきシーン」を明示する
  - 不要な方法は削除または降格（付録・補足扱い）する
- `templates/l1-prompts.md`・`templates/l2-prompts.md` のプロンプトテンプレートを見直す
  - ルールファイル指定を口頭指示から明示的な参照方式に変更する
- 上記変更に伴い `docs/workflow.md` への影響がある場合は合わせて更新する

### やらないこと

- `.claude/rules/*.md` のルール内容の変更（別施策：セッション前提条件・計画レビュー強制）
- Claude Code の設定ファイル（`settings.json`）の変更
- 起動スクリプト（`bash` ラッパー等）の新規作成

## 期待される効果

- セッション開始時の「どの方法で起動するか」の判断コストがゼロになる
- ルールファイルが確実に読み込まれ、セッションごとの動作ばらつきが減る
- プロンプトテンプレートが実態に即した内容になり、コピペしてすぐ使える状態になる

## リスク

| リスク | 影響度 | 対策 |
|--------|--------|------|
| `--rules` フラグの挙動が想定と異なる（ファイルの読み込み方が期待通りでない） | 中 | フェーズ1でフラグの動作を検証し、問題があれば `@ファイルパス` 記法など代替手段を検討する |
| 起動方法の整理で既存の方法2・3が「使えなくなった」と誤解される | 低 | 削除ではなく「サポート外・参考」扱いとして残し、移行理由を明記する |
