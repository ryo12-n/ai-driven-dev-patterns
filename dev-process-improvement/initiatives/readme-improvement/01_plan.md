## リスクと対策

| リスク | 発生確率 | 影響 | 対策 |
|--------|---------|------|------|
| `--rules` フラグが期待通りに動作しない、または存在しない | 中 | 中 | フェーズ1でフラグの動作を確認し、問題があれば `@ファイルパス` 記法や `CLAUDE.md` への読み込み記述など代替手段を選択する |
| README の整理で「方法2・3」を削除することで情報が失われる | 低 | 低 | 削除前に内容を確認し、有用な情報は付録または `docs/` に移動する |
| フェーズ1の調査が発散してルールファイルの内容変更まで踏み込む | 低 | 中 | スコープは README・テンプレートの「記述方式」変更に限定し、ルール内容の変更は `07_issues.md` に記録して別施策へ |
