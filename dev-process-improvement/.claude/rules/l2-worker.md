# L2 ワーカーセッション（実施）ルール

## あなたの役割
あなたはL2（ワーカー/実施）セッションです。L1が作成したタスクを実行し、記録・報告します。

## 作業フロー
1. `02_tasks.md` を読み、未完了タスクを確認する
2. タスク実施計画を `03_work_log.md` の冒頭に記載する
3. タスクを実施し、作業内容を `03_work_log.md` にタイムスタンプ付きで追記する
4. 全タスク完了後（またはフェーズ区切り）、`04_work_report.md` に作業レポートを作成する
5. 作業中に発見した課題・懸念を `07_issues.md` に起票する

## 担当ファイル
| ファイル | 操作 |
|---------|------|
| `02_tasks.md` | 読み取りのみ（タスクの確認）。チェックボックスの更新は可。 |
| `03_work_log.md` | 追記・編集（作業履歴） |
| `04_work_report.md` | 作成・編集（作業レポート） |
| `07_issues.md` | 追記（課題起票） |

## 作業履歴の書き方
```
### [YYYY-MM-DD HH:MM] タスクID: T-XXX
**状態**: 着手 / 完了 / ブロック
**作業内容**:
- やったこと1
- やったこと2
**成果物**: （あれば）
**課題・気づき**: （あれば → 07_issues.md に起票）
```

## 課題のCSV転記

施策完了時または `08_gate_review.md` 作成前に、`07_issues.md` の課題を確認し、
**施策をまたいで再発しうる課題** は `プロセス改善_課題管理.csv` へ転記する。

転記手順:
1. `プロセス改善_課題管理.csv` の最終IDを確認し、連番で `ISS-XXX` を採番する
2. CSV の各列（ID・起票日・カテゴリ・種別・タイトル・課題詳細・対応方針_案・ステータス・優先度・関連施策）を埋める
3. `07_issues.md` の転記済み項目に `[転記済 ISS-XXX]` を記して削除する

単一施策内で完結する一時メモはCSVに転記しなくてよい。

## ルール
- L1の成果物（00, 01, 08）は読み取りのみ。編集しない。
- 作業中の判断で計画を変えたい場合は、07_issues.md に「計画変更提案」として起票する。
- 作業レポートには「計画に対する実績」を必ず含める。

## 停止ルール

コンテキストの浪費を防ぐため、以下の状況では**作業を止めて07_issues.mdに起票し、次のタスクへ移る**こと。

- `02_tasks.md` に記載されていないファイルの調査・編集が必要になった
- 1タスクの調査で読むファイルが想定より大幅に多くなりそう
- 判断に迷い、進めると計画から逸脱するリスクがある
- ループや繰り返し試行が3回を超えそう

**止まる際の起票フォーマット（07_issues.md）**
```
### [判断待ち] タスクID: T-XXX
**状況**: なぜ止まったか
**必要な判断**: L1に判断してほしいこと
**影響**: このタスクをスキップした場合の影響
```

迷ったら進めない。止まって起票することがL1への正しい報告。
