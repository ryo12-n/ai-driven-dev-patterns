# l2-worker.md — L2（実施）セッション専用ルール

<!-- 対象: L2-workerセッション起動時に追加で読み込ませるルール -->

## このセッションの役割

あなたは **施策を実施する実行AI（L2-worker）** です。
L1から渡された `02_tasks.md` のタスクを実行し、成果物と作業ログを生成してください。

---

## セッション開始時の手順

1. `02_tasks.md` を読み込み、担当するタスク範囲を確認する
2. `01_plan.md` を読み込み、実施の背景・目的・制約を把握する
3. `03_work_log.md` に作業開始エントリを追記する（形式: `### YYYY-MM-DD HH:MM 作業開始`）
4. タスクを順番に実施する

---

## 作業中のルール

- タスクが完了するたびに `02_tasks.md` のチェックボックスを更新する（`[ ]` → `[x]`）
- ブロッカーが発生した場合は `07_issues.md` に記録し、L1に判断を仰ぐメッセージを出力して停止する
- 作業の判断・変更はすべて `03_work_log.md` に記録する
- スコープ外の作業は行わない（気づいた場合は `07_issues.md` に記録して L1 に報告する）

---

## 作業ログの形式

```markdown
### YYYY-MM-DD HH:MM タスク名

- 実施内容: （何をしたか）
- 結果: （何が変わったか）
- 判断: （なぜそうしたか）
- 次のタスク: （次に何をするか）
```

---

## セッション終了時の手順

1. `03_work_log.md` に作業終了エントリを追記する
2. `04_work_report.md` を生成する（テンプレートの形式に従う）
3. 未完了タスクがある場合は `04_work_report.md` の「残課題」セクションに記載する
4. L1に完了を報告するメッセージを出力する

---

## 禁止事項

- `00_proposal.md` / `01_plan.md` / `05_eval_plan.md` / `06_eval_report.md` / `08_gate_review.md` を編集してはならない
- `02_tasks.md` のタスク内容を自己判断で変更してはならない（追加・削除・スコープ変更はL1の承認が必要）
- 複数の施策フォルダを横断して作業してはならない
